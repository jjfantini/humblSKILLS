# create-video-transition

Generates a coordinated set of prompts — Prompt A (start image), Prompt B
(end image), Prompt C (video transition in exact 2-second blocks) — and
delivers them in a gorgeous HTML page with live-tweakable settings.

## When to Use

- User wants to animate between two visual states and needs the prompts (not the renders)
- User uploaded a starting image and wants a transition to an edited version as the final frame
- User uploaded an ending image and needs the starting frame inferred backwards
- User describes a transition in words only — all three prompts must be generated from scratch

## Examples

### Example 1: Start image uploaded, want a tweaked final frame

User says: "Here's the starting frame [image.png]. Make a 6-second
transition where the product slowly rotates and the background shifts
from studio white to a soft gradient."

Actions:
1. Read `wiki/workflow/detect/input-mode.md` — mode resolves to `start_only`.
2. Confirm `$VIDEO_LENGTH = 6` from the user's message; default `$VIDEO_ASPECT_RATIO = 16:9`, `$VIDEO_AUDIO = false`, `$VIDEO_FPS = 30`.
3. Read `wiki/prompt/image/end-frame.md` + `wiki/prompt/image/reference-fidelity.md` — generate Prompt B, instructing the image model to use the upload as a visual reference and preserve every asset, material, lighting, and framing attribute except the requested background shift.
4. Read `wiki/prompt/video/block-timeline.md` + `wiki/prompt/video/director-style.md` — generate Prompt C with exactly 3 blocks (0-2s, 2-4s, 4-6s).
5. Read `wiki/output/html/template-fill.md` + `placeholder-tab.md` + `deliver.md` — fill the template (Tab A renders "Provided by user" placeholder; Tabs B and C show generated prompts), write `prompts.html` to the user's cwd, `open prompts.html`.
6. Append to `log.md`.

Result: `prompts.html` opens with 3 tabs; Tab A is a "Provided by user" card; Tabs B and C have copy buttons; the gear icon opens a settings modal for the four video variables.

### Example 2: End image uploaded, generate the start frame

User says: "Here's the final frame [image.png]. I want an 8-second
transition where it zooms in and assembles from a wider angle showing the
full workspace."

Actions:
1. Mode = `end_only`.
2. `$VIDEO_LENGTH = 8` (4 blocks).
3. Generate Prompt A per `wiki/prompt/image/start-frame.md` + `reference-fidelity.md` — the start image must be wider-angle but use identical materials, lighting direction, color palette, and camera system as the uploaded end frame.
4. Generate Prompt C with 4 blocks describing the zoom-in and assembly motion per block.
5. Build HTML — Tab A shows Prompt A; Tab B shows the "Provided by user" placeholder; Tab C shows Prompt C. Open.

Result: The inferred start frame will render as a plausible predecessor to the uploaded end image.

### Example 3: No images uploaded — all three prompts from description

User says: "Give me prompts for a 4-second transition where a smoothie
explodes into its ingredients, glass shatters, liquid flies, freeze-frame
at the end."

Actions:
1. Mode = `neither`.
2. `$VIDEO_LENGTH = 4` (2 blocks).
3. Generate Prompt A (intact smoothie), Prompt B (exploded freeze-frame), Prompt C (2 blocks describing the explosion onset and peak).
4. Build HTML — all 3 tabs populated. Open.

## Troubleshooting

**Generated image drifts from the uploaded reference**
Cause: Prompt B (or A) did not adequately constrain the image model.
Fix: add explicit "use uploaded image as visual reference" language; enumerate each asset, material, and lighting attribute that must be preserved; list only the specific deltas. See `wiki/prompt/image/reference-fidelity.md`.

**Block count in Prompt C does not match `$VIDEO_LENGTH`**
Cause: the length was not confirmed before Prompt C was written, or the block-count rule was not followed.
Fix: re-read `wiki/workflow/elicit/video-length.md` and `wiki/prompt/video/block-timeline.md`. Block count = `floor(length / 2)` full 2s blocks + one 1s tail block if the length is odd.

**HTML page opens but settings modal does not swap the `$VIDEO_*` tokens**
Cause: placeholders were replaced at fill time instead of left as live tokens.
Fix: the default values `{{DEFAULT_VIDEO_LENGTH}}`, `{{DEFAULT_VIDEO_ASPECT_RATIO}}`, `{{DEFAULT_VIDEO_AUDIO}}`, `{{DEFAULT_VIDEO_FPS}}` go in the modal defaults; the literal strings `$VIDEO_LENGTH`, `$VIDEO_ASPECT_RATIO`, `$VIDEO_AUDIO`, `$VIDEO_FPS` must remain in the prompt text for the Apply button to substitute. See `wiki/output/html/settings-modal.md`.

**HTML page does not open in the browser**
Cause: `open` / `xdg-open` unavailable.
Fix: print the absolute path of `prompts.html` so the user can open it manually. See `wiki/output/html/deliver.md`.
