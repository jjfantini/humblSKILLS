## Canonical layout

```
your-skill-name/
|-- SKILL.md           # Required, exact case
|-- scripts/           # Optional - executable code (Python, Bash, ...)
|   `-- process_data.py
|-- references/        # Optional - documentation loaded on demand
|   `-- api-guide.md
`-- assets/            # Optional - templates, fonts, icons used in output
    `-- report-template.md
```

## Hard rules

### SKILL.md naming

- File MUST be exactly `SKILL.md`. Case-sensitive.
- `SKILL.MD`, `skill.md`, `Skill.md` are all rejected on upload.

### Skill folder naming

- kebab-case only.
- No spaces, no underscores, no capitals.

```
# Correct
notion-project-setup
payflow-onboarding
data-viz-csv

# Wrong
Notion Project Setup         # spaces
notion_project_setup         # underscore
NotionProjectSetup           # PascalCase
notion-Project-Setup         # mixed case
```

- The folder name MUST match the `name:` field in the frontmatter.

### No README.md inside the skill folder

- All documentation belongs in `SKILL.md` or under `references/`.
- If you're distributing the skill via GitHub, put the repo-level `README.md` at the repo root, NOT inside the skill directory.

**Incorrect (README inside skill):**

```
my-skill/
|-- SKILL.md
|-- README.md          # wrong location
`-- references/
```

**Correct (README at repo root, never inside a skill):**

```
my-skills-repo/
|-- README.md          # for human visitors browsing GitHub
`-- skills/
    `-- my-skill/
        |-- SKILL.md
        `-- references/
```

## humblSKILL additions

humblSKILLS add `references/` structure on top of the Anthropic layout:

```
references/
|-- _index.md          # Generated by lint.sh
|-- _brain.md          # Brain protocol spec
|-- _template.md       # Wiki concept shape
|-- patterns.md        # Quantified outcomes
|-- decisions.md       # Non-obvious choices
|-- log.md             # Session log
|-- wiki/<context>/<category>/<concept>.md
`-- raw/               # Human-dropped source material
```

This is still fully Anthropic-compliant - it is `references/` extended with a convention for organising long-lived knowledge. Claude can still read any file here on demand.

## Optional vs required

| Path                       | Required | When to include                          |
|----------------------------|----------|------------------------------------------|
| `SKILL.md`                 | yes      | always                                   |
| `scripts/`                 | no       | when deterministic code beats language   |
| `references/`              | no       | when detail would bloat SKILL.md         |
| `assets/`                  | no       | when skill emits templates/brand files   |
| `README.md` (inside skill) | NEVER    | forbidden                                |
