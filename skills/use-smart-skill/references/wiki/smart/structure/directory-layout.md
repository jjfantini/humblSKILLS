## Canonical Directory Layout

Every Smart Skill MUST match this layout. Deviations break the Brain
Protocol (agents can't find what they need) and `scripts/lint.sh`.

**Incorrect (flat layout):**

```
skill-name/
  SKILL.md
  references/
    some-topic.md              # flat, one topic per file
    another-thing.md
  scripts/
    scaffold.sh
```

No brain meta files, no separation of raw vs distilled, no path-based
taxonomy, no self-learning memory.

**Correct (Smart Skill layout):**

```
skill-name/
  SKILL.md                                # Router + Brain Protocol (< 120 lines)
  references/
    # Authoring meta (static)
    _template.md                          # Wiki concept file shape
    _brain.md                             # Authoritative brain spec + naming + writing principles

    # Brain meta (operational memory, LLM-maintained)
    _index.md                             # Auto-regenerated by lint.sh between sentinel markers
    patterns.md                           # Performance memory (append entries)
    decisions.md                          # Reasoning memory (append entries)
    log.md                                # Append-only session log

    # Wiki (distilled knowledge, LLM-owned)
    wiki/
      <context>/<category>/<concept>.md   # One concept per file

    # Raw (immutable sources, human-owned)
    raw/
      <anything-the-human-dropped>.<ext>
      .gitkeep                            # tracks empty dir

  scripts/                                # optional
    scaffold.sh                           # create a new Smart Skill
    lint.sh                               # health-check + regenerate index

  assets/                                 # optional, static files
```

## File Roles at a Glance

| File / Dir                         | Owner | Read on start | Write on end | Notes                                             |
|------------------------------------|-------|---------------|--------------|---------------------------------------------------|
| `SKILL.md`                         | LLM   | yes           | rarely       | Router + Brain Protocol; under 120 lines          |
| `references/_template.md`          | Human | on demand     | rarely       | Concept file shape; static                        |
| `references/_brain.md`             | Human | on demand     | rarely       | Authoritative brain spec + naming + writing rules |
| `references/_index.md`             | LLM   | yes           | on wiki change | Fully auto-regenerated by `lint.sh`             |
| `references/patterns.md`           | LLM   | yes           | when metrics appear | Append-only in practice                     |
| `references/decisions.md`          | LLM   | yes           | on non-obvious choice | Append-only; never delete                 |
| `references/log.md`                | LLM   | last 5 entries| every session| Strict append-only                                |
| `references/wiki/<c>/<cat>/<x>.md` | LLM   | on demand     | on ingest/query synthesis | `sources:` cites raw                 |
| `references/raw/*`                 | Human | on demand     | never (LLM)  | LLM reads but never renames or edits              |
| `scripts/*`                        | Human | never         | never        | Deterministic, non-interactive, idempotent        |
| `assets/*`                         | Either| on demand     | rarely       | Static files referenced by concepts               |

## Why This Layout

- **Brain reads are fast**: index/patterns/decisions/log are 4 files side
  by side, one read each.
- **Wiki loads are cheap**: agents load only the concepts their task needs.
- **Raw is safe**: the user's ground truth can't be corrupted by the LLM.
- **Sources are traceable**: every wiki claim has a `sources:` array
  pointing at raw files.
- **Taxonomy derives from filesystem**: no registry file to drift. Move
  a wiki concept, the taxonomy updates.
- **Lint is mechanical**: path / frontmatter / filesystem checks are
  straightforward walks.
