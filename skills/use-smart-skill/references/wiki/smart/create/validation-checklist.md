## Validation Checklist

Run this before declaring a new Smart Skill ready. `scripts/lint.sh`
automates most items flagged with [auto].

### SKILL.md

- [ ] Under 120 lines (hard max ~200)
- [ ] `name` frontmatter matches the skill directory name
- [ ] `description` includes BOTH what the skill does and WHEN to trigger it
- [ ] `metadata.author` set, `metadata.version` set
- [ ] `compatibility` either filled in with real env requirements OR removed (no TODO placeholder left in shipped skill)
- [ ] Brain Protocol block present and appears BEFORE When-to-Use
- [ ] CCCCC Architecture table present
- [ ] How-to-Use section routes to specific wiki concepts, not embedded content
- [ ] No reference to deleted meta files (`_contexts.md`, `_contributing.md`)

### references/ meta

- [ ] `_template.md` present and reflects the skill's domain
- [ ] `_brain.md` present (covers protocol, naming, writing principles, sources contract)

### Brain meta files

- [ ] `references/_index.md` has `<!-- GENERATED:START -->` and `<!-- GENERATED:END -->` markers [auto]
- [ ] `references/patterns.md` header + entry-shape guide present
- [ ] `references/decisions.md` header + entry-shape guide present
- [ ] `references/log.md` has at least one bootstrap entry

### Wiki concepts

- [ ] Every wiki file lives at `references/wiki/<context>/<category>/<concept>.md` [auto]
- [ ] Every wiki file's `context`/`category`/`concept` frontmatter matches its path [auto]
- [ ] All required frontmatter fields present [auto]
- [ ] `sources:` paths (if any) resolve to files under `references/raw/` [auto]
- [ ] `last_ingested` is a valid ISO date [auto]
- [ ] Each concept file is under 80 lines (aim for 30-60)
- [ ] One concept per file - no tangled concerns
- [ ] Concrete incorrect/correct examples (not abstract advice)
- [ ] Semantic example names (not `foo`/`bar`)

### Raw

- [ ] `references/raw/.gitkeep` present (so empty dir is tracked)
- [ ] No LLM-renamed files (raw is human territory)

### Scripts

- [ ] Scripts referenced by `command:` frontmatter exist and are executable [auto]
- [ ] Scripts are idempotent and non-interactive
- [ ] `scripts/lint.sh` exits 0

### Review (authoring quality)

- [ ] Error-first structure in every concept (problem -> solution)
- [ ] No content duplicated across wiki files
- [ ] Every paragraph justifies its token cost
- [ ] `sources:` populated honestly (no decorative citations)

### Final

- [ ] `scripts/lint.sh` exits 0 with no hard findings
- [ ] `references/_index.md` was regenerated by the latest lint run
- [ ] A bootstrap entry exists in `log.md` noting the skill was created
- [ ] `metadata.version` is set (e.g., `1.0.0`)
