# create-scroll-animation

Takes an MP4 and produces a single-file React scroll-scrubbed canvas hero
component (the Apple-style effect where scrolling plays the video forward
and backward frame-perfect). Frames extract locally; hosting (Vercel Blob,
Supabase Storage, UploadThing) plugs in later via a URL-pattern prop.

## Examples

### Example 1: Next.js project, user provides an MP4

User says: "I have a product rotation video at `./hero.mp4`. Build me a
scroll-scrubbed hero in my Next.js app; put the frames in `public/frames/`."

Actions:
1. Read `wiki/workflow/interview/phase-order.md` — all Phase 1 inputs present; skip Phase 2 (no brand chrome asked).
2. Run `scripts/probe.sh hero.mp4` → duration=6s, 30fps, 1920×1080, H.264.
3. Decide target frame count = 100 (fits <8MB budget at WebP q=80).
4. Run `scripts/extract-frames.sh hero.mp4 /tmp/frames-raw 100` → 100 PNGs at target_fps=16.67.
5. Run `scripts/to-webp.sh /tmp/frames-raw public/frames 80` → 100 WebPs, total 3.8MB.
6. Read `wiki/react/component/scroll-frame-canvas.md`, `wiki/react/component/preload-strategy.md`, `wiki/nextjs/integration/app-router-setup.md`.
7. Read `assets/templates/ScrollFrameCanvas.tsx.tmpl` → substitute `FRAME_COUNT=100`, `FRAME_URL_PATTERN=/frames/frame_{index}.webp`, `FIT_MODE=cover`, `INCLUDE_LOADER_UI=true`, `INCLUDE_REDUCED_MOTION_FALLBACK=true`.
8. Write `components/ScrollFrameCanvas.tsx` (client component, `'use client'` at top).
9. Append to `log.md`.

Result: one new file in the user's repo (`components/ScrollFrameCanvas.tsx`),
100 WebP frames in `public/frames/`. User imports `<ScrollFrameCanvas />`
and adds `npm i framer-motion` as the only new dependency.

## Troubleshooting

**`ffmpeg: command not found`**
Cause: FFmpeg is not installed or not on PATH.
Fix: confirm with the user, then `brew install ffmpeg` (macOS) or `apt install ffmpeg`.

**WebP frames larger than 60KB each**
Cause: `cwebp` missing; fell through to `ffmpeg -c:v libwebp`.
Fix: install `cwebp` (`brew install webp`), re-run `to-webp.sh`. See `wiki/video/optimize/webp-encoding.md`.

**Component re-renders on every scroll tick**
Cause: draw logic reads motion value into state.
Fix: use `useMotionValueEvent(frameIndex, 'change', ...)` instead. See `wiki/react/component/scroll-frame-canvas.md`.

**Blurry canvas on Retina displays**
Cause: missing `devicePixelRatio` scaling.
Fix: set canvas `width/height` to `size * DPR`; keep CSS size unscaled. See `wiki/react/component/scroll-frame-canvas.md`.

**iOS Safari tab reloads mid-scroll**
Cause: decoded frames exceeded ~384MB canvas cap.
Fix: downshift mobile frame resolution and/or reduce count via `connection.effectiveType`. See `wiki/mobile/memory/ios-safari-cap.md`.
